
# CLAUDE.md — bobicord → RelayApp

Этот репозиторий (**bobicord**, «Voice») эволюционирует в **RelayApp**: Discord-подобный апп, где видео-раздача экрана переезжает с LiveKit SFU на **P2P relay-дерево**, а голос/чат/auth/деплой остаются как есть. Полное ТЗ на исполнение: **`docs/Evolution-TZ.md`** — работай по нему, майлстоунами Э0→Э9.

## Стратегия

Эволюция, **не переписывание**. На каждом шаге приложение остаётся рабочим. Голос через LiveKit SFU остаётся (это часть плана). Заменяется только транспорт **видео**.

## Жёсткие инварианты (не нарушать)

1. **Видео не проходит через сервер** (кроме двух осознанных фолбэков P2P-дерева: TURN-relay и **виртуального серверного ретранслятора Э9** — headless vrelay, passthrough RTP без транскода, активируется ТОЛЬКО когда у сирот нет живых relay-кандидатов или зритель явно выбрал «через сервер»; живые пиры всегда предпочитаются (штраф в скоринге), сервер дренирует виртуала при появлении живой ёмкости. Старый LiveKit-путь — см. п.2 — идёт через SFU, это его осознанный компромисс, не нарушение).
2. **Два параллельных источника вещания.** Нативный клиент (Tauri) вещает в **P2P relay-дерево** (H.264, см. п.4). Браузер вещает по **старому пути через LiveKit SFU** (`engine.share()`, VP8) — оставлено намеренно ради совместимости веба, не временный шаг и не к удалению. Зритель при коннекте **динамически** определяет транспорт конкретного стрима (пришёл ли он через tree-сигналинг или через LiveKit-комнату) и подключается соответствующим `VideoTransport` — выбор идёт по факту вещания, не билд-флагом (`VITE_VIDEO_TRANSPORT` не используется для этого).
3. **Браузерный зритель — лист дерева**: не вещает в дерево. Ретранслируют приоритетно нативные зрители (passthrough). **Исключение Э8 (по решению пользователя):** браузер МОЖЕТ ретранслировать как транскод-фолбэк (Chromium re-encode, малая ёмкость) — включено, чтобы дерево ветвилось без нативных зрителей. Приоритет ретрансляторов: **натив (passthrough) > браузер (транскод) > виртуальный сервер (Э9, passthrough-фолбэк)**. (Старое LiveKit-вещание из браузера, п.2, дерева не касается — это отдельный параллельный путь.)
4. **Кодек видео — H.264** для P2P-дерева (нативный вещатель ↔ любой зритель), low-latency, без B-кадров (совместимость натив↔браузер). Старый LiveKit-путь (веб-вещание) остаётся на VP8 — исторический кодек этого пути, не мигрируется.
5. **Микрофон не примешивается к стриму.** Голос — только через голосовой канал (SFU); чтобы говорить, надо быть в голосовом. Стрим = видео + звук игры/системы.
6. **Захват аудио не должен содержать вывод самого RelayApp** (голоса, чужие стримы, служебные звуки) — иначе эхо/петля. Нативно: WASAPI process loopback INCLUDE (только игра) или EXCLUDE (всё, кроме нашего процесса).
7. **Всё кодирование/масштабирование видео — на стороне вещателя** (2K→1080p = рендер сцены в выходном разрешении на GPU). Сервер видео не транскодирует. Нативные зрители-ретрансляторы форвардят RTP как есть (passthrough). **Исключение Э8 (см. п.3):** браузерный relay транскодирует на хоп — осознанный фолбэк, не для нативного пути.
8. **VST/CLAP-обработка голоса — только в нативе**; в вебе fallback на Web Audio (EQ/компрессор/гейт, без плагинов).
9. Задержка видео ≤ 3 c (глубина дерева ≤ 4); голос ≤ 250 мс.
10. Несколько стримов в комнате = несколько деревьев (ключ `streamId`).

## Где что

- `apps/web/src/engine.ts` — ядро медиа/presence/чат (LiveKit). Главная точка правок видео-пути (share/watch/onSub/onSub video).
- `apps/web/src/store.ts`, `components/ServerView.tsx` — UI/состояние.
- `apps/server/index.js` — Express + SQLite + выдача LiveKit-токенов. Сюда добавляется ws-сигналинг дерева + менеджер.
- `docker-compose.yml`, Caddy, `.github/workflows/deploy.yml` — деплой. Сюда добавляются сервисы дерева/coturn.

## Как работать

- Следуй `docs/Evolution-TZ.md` по порядку майлстоунов; не переходи к следующему, пока не выполнены критерии приёмки (AC) текущего.
- Не ломай голос/чат/auth при правках видео.
- Секреты — только через env/`.example`, не хардкодить (WS_URL в `index.js` сейчас захардкожен — вынести в env при первом касании).
- Коммить помайлстоунно; в описании — какие AC проверены.
